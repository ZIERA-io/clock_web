# Claude Code 킥오프 프롬프트

> 아래 블록을 그대로 Claude Code에 붙여넣으세요. `clock.html`(현재 프로토타입)과 `CLAUDE.md`를 같은 폴더에 두고 시작합니다.

---

커스터마이즈 가능한 시계 웹앱을 프로덕션 수준으로 빌드한다. 첨부된 `clock.html`이 동작하는 단일 파일 프로토타입이고, `CLAUDE.md`에 전체 사양·아키텍처·제약이 정리돼 있다. 먼저 두 파일을 모두 읽고 시작할 것.

## 목표
프로토타입의 기능을 유지하면서 **Vite + React + TypeScript** 정적 앱으로 재구성한다. 백엔드 없이 Vercel에 배포 가능해야 하며, 시계 설정값은 URL 해시에 인코딩해 공유 링크로 복원한다.

## 이번 작업 범위 (Phase 1)
1. `npm create vite@latest`로 React+TS 프로젝트 스캐폴딩, 의존성 최소화.
2. 프로토타입 로직을 컴포넌트로 분리:
   - `<ClockStage>` — 전체화면 시계 렌더 (아날로그/디지털)
   - `<AnalogClock>` / `<DigitalClock>`
   - `<SettingsPanel>` — 우측 설정 패널
   - `useClockConfig()` — 설정 상태 + URL 해시 직렬화/복원 훅
3. 설정 스키마, URL 인코딩(base64url, 유니코드 안전), 테마 프리셋, 타임존 처리는 프로토타입 구현을 그대로 이식.
4. **로고 배치 규칙을 반드시 유지**: 아날로그에서 로고는 바늘보다 뒤 레이어, 시계 중심에서 위쪽(12시 방향, viewBox 기준 약 cx=100 / cy=62)에 배치. 정중앙 금지.
5. 항상 전체화면 시계 + 네이티브 Fullscreen API 토글 유지.
6. 한국어 UI 전면 유지, 다크/라이트 테마 전환 유지, iframe·중첩 프레임 사용 금지.

## 하지 말 것
- localStorage/sessionStorage 사용 금지 — 상태는 React state + URL 해시로만 관리.
- 불필요한 UI 라이브러리(거대한 컴포넌트 킷) 추가 금지. 필요한 스타일은 CSS 모듈 또는 단일 스타일시트로.
- 범위 밖 기능(계정, 결제 등) 추가 금지.

## 작업 방식
- 먼저 파일 구조와 설정 스키마 타입(`ClockConfig`)을 제안하고 확인받은 뒤 구현 시작.
- 각 컴포넌트 완성 시 `npm run dev`로 동작 확인하고 스크린샷으로 검증.
- 커밋은 기능 단위로 작게. 한국어 커밋 메시지.
- 막히면 추측하지 말고 질문할 것.

확인했으면 제안할 파일 구조와 `ClockConfig` 타입부터 보여줘.
