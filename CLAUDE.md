# CLAUDE.md — 시계 웹앱

커스터마이즈 가능한 시계를 디자인하고, 백엔드 없이 고유 링크로 공유하는 정적 웹앱.

## 1. 현재 상태

`clock.html` — 동작하는 단일 파일 프로토타입(바닐라 JS + SVG). 모든 핵심 기능이 이미 구현돼 있다. 프로덕션 작업은 이 파일을 React+TS로 재구성하는 것이며, **기능과 동작은 프로토타입을 기준 진실(source of truth)로 삼는다.**

## 2. 핵심 요구사항

- 시계 종류 전환: 아날로그 / 디지털
- 시계는 **항상 전체화면**으로 화면 중앙에 꽉 차게 렌더. 네이티브 Fullscreen API 토글 버튼 제공.
- 중앙 로고 설정: 텍스트·이모지 / 이미지 URL / 파일 업로드(Base64) 세 방식.
  - **로고 배치(중요):** 아날로그에서 로고는 **시계 바늘보다 뒤 레이어**에 놓이고, 시계 **중심에서 위쪽**(12시 방향, 워치 다이얼 스타일)에 위치. 정중앙 배치 금지. SVG viewBox(0 0 200 200) 기준 로고 시각 중심은 약 `cx=100, cy=62`.
- 고유 링크: 설정값 전체를 URL 해시에 인코딩. 링크를 열면 그 시계가 그대로 복원되고 설정 패널은 자동으로 숨겨진다. 서버·DB 불필요.
- 부가 기능: 테마 프리셋 + 6색 개별 커스터마이징 / 초침·날짜·아날로그 숫자 토글 / 24시간제 / 디지털 폰트 선택 / 타임존 / 제목 표시 / 무조작 시 컨트롤 페이드아웃.

## 3. 설정 스키마 (`ClockConfig`)

```ts
interface ClockConfig {
  type: 'analog' | 'digital';
  colors: {
    bg: string;      // 배경
    face: string;    // 시계판 테두리
    tick: string;    // 눈금/숫자
    hand: string;    // 시·분침
    accent: string;  // 초침/강조
    text: string;    // 디지털 텍스트
  };
  logo: {
    kind: 'none' | 'text' | 'url' | 'image';
    value: string;   // text/url 문자열 또는 image data URL
  };
  seconds: boolean;
  date: boolean;
  numerals: boolean; // 아날로그 숫자 표시
  h24: boolean;      // 디지털 24시간제
  font: string;      // 디지털 폰트 family
  tz: string;        // 'local' 또는 IANA 타임존
  title: string;
}
```

## 4. URL 인코딩

- 직렬화: `JSON.stringify(config)` → UTF-8 안전 base64url(`+`→`-`, `/`→`_`, `=` 제거).
- URL 형태: `https://도메인/#c=<encoded>`.
- 복원: 로드 시 `location.hash`에서 `c=` 파싱 → 디코드 → 기본값과 병합. 실패 시 기본 설정으로 폴백.
- 업로드 이미지는 Base64가 URL에 통째로 들어가 링크가 길어진다. ~1900자 초과 시 사용자에게 경고하고 이미지 URL 사용을 권장.

## 5. 기술 스택 / 배포

- Vite + React + TypeScript.
- 스타일: CSS 모듈 또는 단일 스타일시트. 무거운 UI 킷 금지.
- 배포: Vercel 정적 호스팅. 빌드 산출물만 서빙되면 됨.

## 6. 확장 (Phase 2, 선택)

업로드 이미지 때문에 링크가 길어지는 문제를 줄이려면 단축 링크 백엔드를 붙인다.
- Vercel KV(또는 Supabase)에 설정 JSON 저장 → 짧은 ID 발급 → `/#s=<id>`로 복원.
- 이건 백엔드가 필요하므로 명시적으로 합의된 뒤에만 착수.

## 7. 코딩 규칙 / 제약

- **localStorage/sessionStorage 사용 금지.** 상태는 React state + URL 해시로만.
- iframe·중첩 프레임 금지. 전체 뷰포트 레이아웃.
- UI는 한국어 전면 적용.
- 다크/라이트 테마 전환은 전역 영속 동작.
- 범위 밖 기능(계정/결제/분석 등) 임의 추가 금지.
- 추측 금지 — 사양이 불명확하면 질문.
- 직접적이고 간결하게. 불필요한 옵션 나열·과잉 설명 지양.
